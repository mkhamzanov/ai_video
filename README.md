# AI-video channel — движок образовательных Reels про ИИ и ML

Канал коротких вертикальных видео (1080×1920, 9:16, под Reels/Shorts), которые
**простым языком объясняют ИИ и машинное обучение** новичкам. Ролики собираются
кодом в [Remotion](https://www.remotion.dev/) (React → видео), звук — собственный
синтезированный SFX-пак, музыка и лицо автора накладываются при монтаже.
**Озвучка (RU + KZ) генерируется автоматически** голосом Brian (ElevenLabs) и
кладётся в тайминг субтитров — см. [«Озвучка»](#озвучка-авто-elevenlabs).

> ⚙️ **Этот репозиторий — только ДВИЖОК:** шаблон, дизайн-система, SFX-пак,
> пайплайн озвучки и один пример-заготовка (`example`). **Сами ролики (идеи,
> сценарии, тексты, manim-анимации) хранятся локально и в git НЕ попадают** —
> см. [«Движок и локальный контент»](#движок-и-локальный-контент).

## Что внутри

```
video/                       ⚙️ = в git (движок)   🔒 = локально (вне git)
  linreg/            ← Remotion-проект
    src/
      template/      ⚙️ переиспользуемые блоки (Reel, Title, Captions, Formula, Sound, Music…)
      lib/           ⚙️ бренд, шрифты
      blocks/_example/ ⚙️ пример-заготовка ролика (каркас без контента)
      blocks/*       🔒 реальные ролики (код, сцены, тексты) — на диске, вне git
      Root.tsx       ⚙️ реестр движка (только `example`)
      Root.local.tsx 🔒 твой реальный реестр композиций
      TEMPLATE.md    ⚙️ ⭐ ИНСТРУКЦИЯ как делать ролик (стиль, темп, субтитры, звук)
    public/          ⚙️ ассеты (fonts/, sfx/, music/, avatar.example.jpg)
      avatar.jpg     🔒 твоя личная финальная фотка (рендерится в EndCard)
  scripts/gen_sfx.py ⚙️ генератор SFX-пака (Python)
  scripts/voiceover/ ⚙️ автоозвучка (make.py); captions/*.kk.json — 🔒 (кроме _example)
  public/sfx/        ⚙️ мастер SFX-пак (59 звуков, .wav + .mp3) + README
  .env               🔒 секреты (ключ ElevenLabs)
  manim/             🔒 математические анимации (Manim, Python)
  shots/ aishots/ reelshots/  🔒 исходные кадры/картинки роликов
  *.mp4              🔒 отрендеренные ролики (выход рендера)
  media/             🔒 рабочий кэш Manim
```

> **Главный документ для создания видео — [`linreg/src/TEMPLATE.md`](linreg/src/TEMPLATE.md).**
> Там правила стиля (тёмная тема + жёлтый акцент, шрифт IgraSans), темп и
> читаемость субтитров, звуки, каркас новой композиции. Читай его перед тем, как
> делать новый ролик.

## Движок и локальный контент

Репозиторий намеренно содержит **только движок** — так идеи и сценарии роликов не
утекают. В git: шаблон (`template/`, `lib/`), дизайн-система, шрифты, SFX-пак,
музыка, пайплайн озвучки и один пример `blocks/_example/`.

**Вне git (только на твоём диске), см. `.gitignore`:**
- `linreg/src/blocks/*` — код реальных роликов (кроме `_example`);
- `linreg/src/Root.local.tsx` — твой реальный реестр композиций;
- `linreg/public/avatar.jpg` — твоя личная финальная фотка (рендерится в `EndCard` в конце ролика);
- `scripts/voiceover/captions/*.kk.json` — казахские переводы (кроме `_example`);
- `manim/`, `shots/ aishots/ reelshots/`, `*.mp4` — анимации, кадры, рендеры.

**Финальная фотка.** `EndCard` показывает `public/avatar.jpg` — это личный файл, в git
его нет. В репозитории лежит только плейсхолдер `avatar.example.jpg` (его использует
пример `example`). Положи свою фотку в `linreg/public/avatar.jpg` — реальные ролики
подхватят её автоматически (`<EndCard/>` по умолчанию берёт `avatar.jpg`).

**Как делать новый ролик:** скопируй `blocks/_example/` в `blocks/<имя>/`, наполни
сценами и субтитрами (`timeline.ts`), зарегистрируй композицию в `Root.tsx` (или
веди свой `Root.local.tsx`). Файлы под `blocks/*` автоматически остаются локальными.

> Движок собирается и рендерит `example` из коробки. Реальные ролики видны в
> Remotion-студии только на твоей машине, где есть локальные `blocks/*`.

## Установка

Нужны **Node.js 18+** (рекомендуется LTS) и, опционально, **Python 3.10+** —
только если будешь пересобирать звуковой пак.

```bash
cd linreg
npm install
```

Для генерации SFX (необязательно, пак уже лежит в репозитории):

```bash
pip install numpy        # gen_sfx.py использует numpy
```

## Запуск (превью в студии)

```bash
cd linreg
npm run dev              # = remotion studio, откроется в браузере
```

Превью с направляющими (зона для лица автора снизу + зона «шторки» айфона сверху):

```bash
cd linreg
npx remotion studio --props=./guide-props.json
```

## Рендер видео

```bash
cd linreg
npx remotion render <id> ../<имя>.mp4 --log=error --timeout=120000
```

Готовый `.mp4` появляется в корне `video/` (в git не хранится). Пример-заготовка:

```bash
cd linreg
npx remotion render example ../example.mp4 --log=error --timeout=120000
```

Свои ролики (из локальных `blocks/*`) рендерь так же по их `id`, предварительно
зарегистрировав их локально — см. [«Движок и локальный контент»](#движок-и-локальный-контент).

## Озвучка (авто, ElevenLabs)

Стандартный конвейер: **готовый ролик → голос Brian в тайминг субтитров → микс
поверх фоновой музыки (Radiohead, уже в mp4) → `<блок>.<lang>.voiced.mp4`.**
Клонирование голоса не используется — Brian звучит одинаково на RU и KZ.

Формат — вертикаль 1080×1920 под Instagram Reels: субтитры вверху (зона `subtitleTop`),
нижние ~30% пустые (совпадают с зоной UI Инсты: подпись/аудио/кнопки), поэтому
контент не перекрывается интерфейсом.

Настройка (один раз): скопируй `.env.example` → `.env` и впиши ключ ElevenLabs.
Голос/модель по умолчанию — Brian + `eleven_v3` (**единственная модель с казахским**).

```bash
# русская озвучка (текст субтитров берётся из timeline.ts как есть):
python scripts/voiceover/make.py --block clickhouse --lang ru

# казахская: нужен перевод subтитров scripts/voiceover/captions/<блок>.kk.json
# (строки в том же порядке и количестве, что CAPS в timeline.ts):
python scripts/voiceover/make.py --block clickhouse --lang kk
```

Как работает: скрипт читает `start`-кадры и текст субтитров из
`linreg/src/blocks/<блок>/timeline.ts`, озвучивает каждую реплику, подгоняет её
длину под слот до следующей реплики через `atempo` (≤1.7×) и ставит на нужный
кадр → **голос синхронен субтитрам**. Оригинальная дорожка (музыка+SFX) приглушается
под голос (`--music-vol`, по умолч. 0.35).

> **KZ-нюанс.** Скрипт кладёт казахский голос поверх видео, где субтитры пока
> русские. Чтобы субтитры тоже были на казахском, ролик нужно перерендерить из
> Remotion с казахским треком CAPS (i18n-версия шаблона — следующий шаг).
>
> **Тариф.** free-аккаунт ElevenLabs = 10 000 симв. и без коммерческой лицензии;
> для всего сериала RU+KZ нужен платный план.

## Manim — математические анимации (локально)

Часть роликов с «живой» математикой делается на [Manim Community](https://www.manim.community/)
(Python), а не на Remotion. **Код `manim/` хранится локально (вне git)** — это тоже
контент. Общий рецепт: вертикаль `-r 1080,1920`, LaTeX не нужен (математика юникодом
в `Text`), звук/музыка накладываются системным ffmpeg. Подробности — в комментариях
внутри локальных `manim/*.py`.

> Гочи Manim здесь: LaTeX не установлен — математику набираем юникодом в `Text`
> (`β₁`, `λ`, `‖·‖`, `²`), а не через `MathTex`. Для портрета обязательно
> `config.frame_height=8; config.frame_width=4.5` + рендер с `-r 1080,1920`.

## Звуковой пак (SFX)

59 синтезированных, royalty-free звуков в `public/sfx/` (зеркало — `linreg/public/sfx/`).
Семейства и применение — в [`public/sfx/README.md`](public/sfx/README.md).
В коде звуки подключаются через `template/Sound.tsx` (логические имена → файлы пака).

Перегенерировать пак:

```bash
python scripts/gen_sfx.py        # стирает и пересобирает public/sfx (нормализация −19 LUFS)
```
