# Шаблон ролика (Reels) — как делать видео

Единый стиль: тёмная тема redstat, шрифт **IgraSans**, фоновая музыка на весь ролик,
мягкие звуки по смыслу, **формулы только через LaTeX**, субтитры повыше/поменьше,
снизу — место для лица автора, в конце — пустое место под подписку (автор вставит сам).

> **ЗОНА ЛИЦА — ГЛАВНОЕ ОГРАНИЧЕНИЕ.** Нижние **30 % кадра** (y ≥ `ZONE.faceTop` = 1344)
> зарезервированы под talking-head автора и ДОЛЖНЫ оставаться пустыми. `Stage` сам обрезает
> весь контент по верхним 70 % (бокс высотой 1344, `overflow:hidden`), поэтому всё, что
> выходит ниже, просто срежется. Держи контент в `ZONE.contentTop…contentBottom`
> (560…1300). Центрированные через `AbsoluteFill` сцены центрируются на y≈672 — это нормально.

> **ЗОНА ШТОРКИ АЙФОНА — ВЕРХНЕЕ ОГРАНИЧЕНИЕ.** Верхние **20 % кадра**
> (y ≤ `ZONE.notchBottom` = 384) перекрываются Dynamic Island / статус-баром айфона —
> туда тоже НЕ кладём важный контент. В отличие от зоны лица, `Stage` это НЕ обрезает
> (фон/декор там допустимы), но текст и графики держи ниже 384. Для таких роликов:
> субтитры опускают под шторку — `<Captions track={CAPS} top={ZONE.notchBottom + 16} />`
> (у `Captions` есть необязательный проп `top`, по умолчанию `ZONE.subtitleTop`=300);
> большой `Title` (он у y=70, под шторкой) заменяй компактным локальным «кикером»
> на уровне ~560; весь контент — в полосе **[580, 1320]** (между шторкой и лицом).
> В превью с `guide=true` сверху рисуется полосатый гайд «ШТОРКА · 20%».
> Пример такого ролика — `blocks/rocauc/`. Старые ролики шторку НЕ учитывают
> (это opt-in, `titleTop/subtitleTop` не меняли) — добавляй её только в новых.

> **ГЛАВНОЕ ПРАВИЛО — ПЛОТНО И БЫСТРО, БЕЗ ПАУЗ.** Ролики ОЗВУЧИВАЮТСЯ, поэтому темп должен быть
> *плотный и динамичный*: реплики идут почти встык, тишины между ними почти нет — «напичкано»,
> живо, быстро. Дели мысль на короткие фразы (≤5 слов) и ставь их одну за другой; ясность даёт
> **голос + короткая фраза**, а не длинная пауза. НЕ растягивай слоты и не оставляй «мёртвый»
> хвост в конце сцены. (Раньше тут было «не торопиться / медленно и подробно» — это УСТАРЕЛО:
> с озвучкой длинные паузы = провисание. Точные цифры слотов — в разделе «Темп и субтитры».)

## Структура

```
src/
  template/        ← переиспользуемые блоки (импортировать отсюда: "../../template")
    format.ts      VIDEO (1080×1920, 30fps) + ZONE (зоны кадра)
    Reel.tsx       ОБЁРТКА ролика: фон + контент + фоновая музыка (используй вместо Stage)
    Stage.tsx      фон (+ guide=true рисует зоны: лицо снизу + шторку сверху)
    Title.tsx      кикер + заголовок сверху (НЕ для роликов со шторкой — он у y=70)
    Caption.tsx    Captions — субтитры (выше центра; проп top опускает их под шторку)
    Formula.tsx    LaTeX-формула через KaTeX  ← ВСЕ формулы только так
    Sound.tsx      SoundTrack — мягкие звуки по абсолютным кадрам
    Music.tsx      BackgroundMusic — radiohead на весь ролик (уже внутри Reel)
    EndCard.tsx    пустой финал (место под аккаунт/подписку — автор вставит в монтаже)
    index.ts       один импорт
  blocks/{ai,math,infographics}/   ← сами видео
```

## Правила (обязательно)

0. **ТЕМА: чёрный фон + жёлтый и белый шрифт.** Акцент/подсветка — жёлтый (`C.yellow`),
   основной текст — белый (`C.text`). Если фон светлый/белый — текст чёрный (`C.ink`).
   Данные на графиках могут быть других цветов (нужны различимые оттенки).
1. **Обёртка — `<Reel durationInFrames={TOTAL} guide={guide}>`** (не голый `<Stage>`).
   Внутри Reel играет фоновая музыка `public/music/radiohead.mp3` **на уровне оригинала**
   `main_music_radiohead.MOV` (volume = 1, без приглушения; только микро-fade на стыках).
   Звуковые эффекты (`SoundTrack`) — наоборот, **тихие** (volume ≈ 0.3–0.4).
2. **Шрифт — всегда IgraSans** (`FONT`). В тексте никаких системных шрифтов.
3. **Формулы — ТОЛЬКО LaTeX** через `<Formula tex="y = k \\cdot x + b" size={70} color={C.red} />`.
   Никаких формул «руками» из span-ов.
4. **Максимум анимации** — всё через `spring()` / `interpolate()`. Анимации тоже не торопи:
   появления плавные, без резких морганий; важный элемент держи на экране, не убирай сразу.
5. **Очень простой язык + ПОДРОБНО** — короткие фразы, по одной мысли на реплику. Каждую
   идею разжёвывай: сначала назови, потом объясни «что это значит» и дай пример из жизни.
   Не глотай шаги — лучше 3 простые реплики, чем 1 умная.
6. **Мягкие звуки везде** (`SoundTrack`, громкость по умолчанию тихая). Бери звуки
   **только из существующих семейств** `public/sfx`: `back, blip, bloop, click, error,
   hover, keypress, pop, tap, tick, toggle` (полный список — в `public/sfx/README.md`).
   По смыслу: `pop` — появление, `back` — смена сцены, `blip` — важный факт/акцент,
   `tap`/`tick` — мелочь, `bloop` — число/итог, `error` — ошибка/предупреждение.
   Логические имена `SfxName` (в `Sound.tsx`) маппятся на эти файлы:
   `pop, tap, tink, tick, error, whoosh, chime, success, bloop`. Семейства
   `bell/chime/confirm/notify/select/success/swipe/whoosh` и любые `_preview_*` УДАЛЕНЫ —
   не использовать (их в папке нет).
7. **Субтитры — выше и компактнее** (`Captions`, зона `ZONE.subtitleTop`).
   Низ кадра (`ZONE.headTop…headBottom`) НЕ занимать — туда автор наложит лицо.
   Субтитры — главный канал смысла, держи их на экране долго (см. «Темп» ниже).
8. **В конце — `<EndCard guide={guide} />`** (пустое место под подписку). Никаких кнопок
   «подписаться»/стикеров в шаблоне — автор добавит свой аккаунт при монтаже.

## Темп и субтитры (плотно и быстро — минимум пауз)

Всё в кадрах при **30 fps**. Цель — *плотная непрерывная речь*: реплики идут почти встык,
тишины между ними почти нет. Голос короткой фразы (3–5 слов) ≈ 1.5–3 с — слот делаем ПОД это,
не больше.

**Длительность одной реплики субтитра** (`Cap.end - Cap.start`):
- на глаз **`dur ≈ max(45, длина_текста × 2.8)` кадров** (≈ 10–11 знаков/сек — быстрый живой темп).
  Это **вдвое короче** старого правила ×4 — то было для роликов БЕЗ голоса, больше НЕ применяем.
- **минимум ~45 кадров (1.5 с)** даже для пары слов; длинную реплику лучше РАЗРЕЗАТЬ, чем растягивать.
- зазор между репликами — **6–8 кадров (~0.2 с)**, НЕ 10–15. Ровно чтобы глаз заметил смену.
- **никакого «мёртвого» хвоста**: последняя реплика сцены кончается за ~10 кадров до конца сцены.
  Сцена = интро ~10 + сумма слотов подряд + хвост ~12.

**Размер реплики:** ≤ **5 слов / ~34 символа**. Длинную мысль дели на 2–3 БЫСТРЫЕ реплики
подряд — так плотнее, чем одна длинная. Одна реплика = одна мысль.

**Длительность сцены:** ровно столько, сколько нужно её репликам впритык (обычно **5–9 с**).
Не добивай сцену пустотой ради «медленно». Хук/финал — ещё короче.

**Переходы между сценами** — быстрые: звук `back` на стыке, контент въезжает за ~15–20 кадров.
Между фоном и первым субтитром — ~6–10 кадров «вдоха», не больше.

**Общая длина ролика:** сколько выйдет при плотном темпе, обычно **50–70 с**. Короче = лучше,
если смысл не теряется.

> **Проверка после озвучки** (главная): глянь лог `make.py` — `tempo` должен быть **~1.0–1.3**
> (голос влезает в слот без спешки) и `slot − dur` маленькое (мало тишины). Если `tempo > 1.4` —
> реплика длинная, РЕЖЬ её на две. Если `slot − dur` большое (> 1 с) — слот раздут, укороти `end`
> или подвинь следующую реплику раньше. Субтитр не должен «мелькать», но и висеть в тишине — тоже.

## Озвучка (RU + KZ, авто)

Субтитры — это одновременно **сценарий голоса**. После рендера ролик озвучивается
автоматически голосом Brian (ElevenLabs) в тайминг субтитров и микшируется поверх
музыки — см. [README → Озвучка](../../README.md#озвучка-авто-elevenlabs).

Поэтому пиши `text` в `CAPS` так, чтобы его было **не стыдно произнести вслух**:
законченная мысль, без «телеграфности». Слот под голос = время до следующей реплики,
поэтому держи слоты КОРОТКИМИ (см. «Темп»: `dur ≈ длина × 2.8`, зазор 6–8 кадров) — тогда
речь звучит плотно, без провисаний.

**Символы и числа — голос читает криво** (`₸`, `3.5`, `½`, `×`, `−`, `+` → «три поинт файв»,
тенге не произносит). make.py авто-нормализует (`ru_speech`: премап символов + `rutextnorm`
разворачивает числа словами; `pip install rutextnorm num2words`). Для точной формулировки
у реплики есть необязательное поле **`say`**: на экране остаётся красивый `text` («+250₸», «3.5»),
а голос берёт `say` («плюс двести пятьдесят тенге», «три с половиной»). `say` перебивает
авто-нормализацию — используй его для денег, дробей и всего, что должно звучать однозначно.

Для казахской версии на каждый блок заводится перевод
`scripts/voiceover/captions/<блок>.kk.json` — строки в **том же порядке и количестве**,
что `CAPS`. Технические термины (SQL, ClickHouse, RAG, LLM) и бренды не переводим.

## Каркас нового видео

**Старт — копия `blocks/_example/`** (готовый каркас: `timeline.ts` + компонент).
Переименуй, замени сцены/тексты, зарегистрируй в `Root.tsx`. Реальные ролики под
`blocks/*` остаются **локально, вне git** (см. [README → Движок и локальный контент](../../README.md#движок-и-локальный-контент)).

```tsx
export const TOTAL = /* сумма длительностей сцен */;

export const MyVideo: React.FC<{ guide?: boolean }> = ({ guide = false }) => (
  <Reel durationInFrames={TOTAL} guide={guide}>
    <Series>
      {/* сцены: <Title/>, контент в зоне ZONE.contentTop..contentBottom, <Formula/> */}
      <Series.Sequence durationInFrames={150}><EndCard guide={guide} /></Series.Sequence>
    </Series>
    <Captions track={CAPS} />
    <SoundTrack cues={CUES} />
  </Reel>
);
```

Регистрация в `src/Root.tsx` внутри `<Folder>` (имена папок — только ASCII).
Превью: `npx remotion studio`. Рендер: `cd linreg && npx remotion render <id> ../out.mp4 --log=error --timeout=120000`.
Зона лица + место подписки видны в превью с `--props=./guide-props.json`.

**Финальная фотка (`EndCard`).** По умолчанию `<EndCard/>` берёт `public/avatar.jpg` —
это **личный файл, в git его нет** (в репо лежит только плейсхолдер `avatar.example.jpg`).
Положи свою фотку в `linreg/public/avatar.jpg`. Для примера/без личной фотки передавай
`<EndCard avatar="avatar.example.jpg" />`. См. [README → Движок и локальный контент](../../README.md#движок-и-локальный-контент).
